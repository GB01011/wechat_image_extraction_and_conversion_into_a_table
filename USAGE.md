# 使用手册

## 快速上手

### 1. 准备工作

在启动程序前，确保：

✅ 微信已打开并登录  
✅ 配置文件 `config/settings.yaml` 已修改（填入真实的好友名和 API Key）  
✅ 虚拟环境已激活：`interpreter\Scripts\activate`  

---

### 2. 启动程序

```bash
python main.py
```

程序会持续监听指定的微信对象，当有新消息时自动处理。

---

### 3. 停止程序

在命令行中按 `Ctrl + C`，程序会优雅地退出。

---

## 消息处理细节

### 识别的图片格式

程序支持以下表格格式：

| 格式 | 说明 | 例子 |
|------|------|------|
| **横向表格** | 行为产品，列为属性 | 品名 规格 价格...|
| **纵向表格** | 列为产品，行为属性 | 品名: 钢管... |
| **不规则格式** | 任意排列的表格 | 混合排列 |
| **文字形式** | 纯文本表格 | 每行一条数据 |

### 识别的文字格式

程序会自动识别以下文字形式的数据：

1. **多行分隔**（用空格或制表符分隔）：
```
品名    规格      价格
钢管    Φ10*1.5   150元/支
螺栓    M8*20     2元/个
```

2. **逐行列举**（包含关键词）：
```
品名: 钢管
规格: Φ10*1.5
产地: 天津
价格: 150元/支
```

3. **自由格式**（包含表格关键词）：
```
这是一份报价单
钢管、Φ10*1.5mm、碳钢、天津、库房A、150元一支
螺栓、M8*20mm、不锈钢、浙江、库房B、2元一个
```

### 不被识别的情况

以下情况会被跳过：

❌ 普通聊天文本（不含表格关键词）  
❌ 表情包或贴纸  
❌ 视频或语音消息  
❌ 链接或小程序卡片

## 生成的 Excel 格式

### 标准输出格式

所有生成的 Excel 都遵循以下格式：

| 列号 | 字段 | 说明 | 例子 |
|------|------|------|------|
| A | 品名 | **产品名称**（必填） | 钢管 |
| B | 规格 | 产品规格 | Φ10*1.5 |
| C | 材质 | 产品材质 | 碳钢 |
| D | 产地 | 生产或来源地 | 天津 |
| E | 仓库 | 存储位置 | 库房A |
| F | 价格 | 产品价格 | 150元/支 |

### 表格样式

✨ **自动格式化**：
- 蓝色表头，白色字体
- 方形边框，美观对齐
- 表头冻结，便于滚动查看
- 列宽自动调整

### 文件命名规则

```
表格_{来源}_{时间戳}.xlsx

例如：
表格_销售部-王经理_20240303_103045.xlsx
表格_文件传输助手_20240303_110230.xlsx
```

## 查看日志

### 日志位置

```
logs/system_YYYY-MM-DD.log
```

例如：`logs/system_2024-03-03.log`

### 日志内容示例

```
2024-03-03 10:30:45 | INFO     | ✓ 配置加载成功
2024-03-03 10:30:46 | INFO     | 正在初始化各模块...
2024-03-03 10:30:47 | INFO     | ✓ 微信连接成功
2024-03-03 10:30:48 | INFO     | 📬 收到 1 条待处理消息
2024-03-03 10:30:49 | INFO     | ✓ 解析成功，提取到 2 条记录
```

### 日志级别

| 级别 | 符号 | 使用场景 |
|------|------|--------|
| DEBUG | 🔍 | 详细的诊断信息（调试用） |
| INFO | ✓ | 普通信息（程序运行状态） |
| WARNING | ⚠ | 警告（可能需要注意） |
| ERROR | ❌ | 错误（处理失败但程序继续） |
| CRITICAL | 🚨 | 严重错误（程序可能无法继续） |

### 修改日志级别

在 `config/settings.yaml` 中修改：

```yaml
logging:
  level: "DEBUG"  # 改为 DEBUG 可看到更详细的信息
```

## 常见提问

### Q1：程序一启动就退出了？

**A**：检查 `config/settings.yaml` 是否有格式错误或配置不完整。查看日志文件获取详细信息。

### Q2：微信消息没有被采集？

**A**：
- 检查好友/群名是否在 `listen_list` 中
- 检查好友/群名拼写是否与微信完全相同
- 查看日志中是否有 "成功添加监听" 的消息

### Q3：图片识别不准确？

**A**：
- 启用图片预处理：`enable_preprocessing: true`
- 拍摄或截图时确保表格清晰，对比度高
- 考虑更换更强大的模型（如 GPT-4V）

### Q4：Excel 文件损坏或打不开？

**A**：
- 关闭已打开的相同文件
- 删除损坏的文件，重新生成
- 确保 `data/output_excel/` 目录的路径没有中文（某些 Excel 版本可能有兼容性问题）

### Q5：可以同时监听多个对象吗？

**A**：可以。在 `listen_list` 中添加多个好友或群名：

```yaml
wechat:
  listen_list:
    - "好友1"
    - "好友2"
    - "群聊1"
    - "群聊2"
```

### Q6：可以删除临时图片吗？

**A**：可以。处理完成后，`data/temp_images/` 中的图片可以安全删除。Excel 文件在 `data/output_excel/` 中，请勿删除！

### Q7：怎样修改 Excel 输出路径？

**A**：编辑 `config/settings.yaml`：

```yaml
output:
  excel_dir: "C:/my_output"  # 改为你想要的路径
```

## 数据质量控制

### 数据验证规则

1. **必填字段检查**：品名不能为空
2. **空字段限制**：不超过 4 个空字段（共 6 个字段）
3. **去重检查**：相同品名和规格的记录只保留一条
4. **字符清洗**：自动移除前后空格

### 数据质量评分

程序计算所有记录中填充字段的比例作为"质量评分"：

- **100%**：所有字段都有值
- **66%**：4/6 字段有值
- **33%**：2/6 字段有值

质量评分越高，数据越完整。
